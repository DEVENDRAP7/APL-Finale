The Challenge
What breaks at 72,000 people

Every large event faces the same systemic failures — and they all compound each other at scale.

🚶‍♂️

Crowd Bottlenecks

Entry gates and concourses become dangerously congested with no real-time intelligence to redistribute flow.

⏳

Long Wait Times

Concession queues of 15–25 minutes with no dynamic staffing response or attendee guidance to alternatives.

📻

Siloed Staff Comms

Security, medics, ushers, and logistics operate on separate channels with no unified coordination layer.

🚨

Slow Emergency Response

Medical incidents and safety threats take minutes to escalate — time that costs lives in critical situations.

🗺️

No Personal Guidance

Attendees navigate 72,000-person venues alone — no smart assistant for seat finding, food orders, or accessibility.

🚗

Post-Game Chaos

Exit surges overwhelm parking lots and transport with no predictive management or dynamic routing.

The 3 AI Agents

Only the decisions that genuinely require AI reasoning. Everything else is fast, rule-based code that runs instantly at zero API cost.

💬
Attendee Experience Agent
gemini-2.5-flash · Streaming

Handles every fan's personal query in natural language — seat finding, accessible routes, food ordering, lost & found. Streams responses in real time to the mobile app via WebSocket.

lookup_seatget_queue_waitfind_nearest_amenityplace_food_orderrequest_accessibility_help
🚨
Emergency Response Agent
gemini-2.5-pro · Adaptive Thinking

Safety-critical reasoning for medical incidents, security threats, fires, and evacuations. Uses Opus 4.7 because every emergency is unique — no rule covers every combination of simultaneous incidents.

get_venue_stateget_evacuation_routesactivate_pa_announcementcontact_emergency_servicestrigger_zone_evacuationdispatch_medical_team
👁️
Vision Anomaly Agent
gemini-2.5-flash · Vision

Interprets flagged camera frames using Gemini Vision. YOLOv8 runs free on all 96 cameras every 5 seconds. When anomaly_score exceeds 0.7, Gemini Vision determines exactly what's happening.

analyse_frameassess_safety_riskrecommend_action

Architecture
How it works

Five layers from physical hardware to real-world action. The rule engine handles 80% instantly — AI agents handle the 20% rules can't.

1

Physical Sensors

284 IoT sensors + 96 CCTV cameras publish data every 2–5 seconds via MQTT and RTSP streams.

2

Ingestion Pipeline

Raw sensor data is normalised, threshold-gated, and enriched with zone metadata before entering the system.

3

Rule Engine

80% of decisions — crowd routing, queue alerts, staff dispatch, parking — are instant rule-based code. Zero API cost.

4

AI Agents

3 Gemini agents handle what rules can't: natural language chat, novel emergency reasoning, visual anomaly interpretation.

5

Real-World Action

PA broadcasts, digital signage, staff dispatch, mobile notifications, and emergency calls — all automated.
